# 全局极客终端与 WebSocket 透传架构设计

## 设计初衷
在现代 AI IDE（如 Antigravity 注入环境）中，传统的“画 UI（如树形目录、Git 状态栏）”不仅开发维护成本高，且往往不如直接使用原生命令行工具（如 `yazi`, `lazygit`）强大。
因此，我们摒弃重型 UI 的开发，转向构建一个**贯穿浏览器与本地系统的全局 Web 终端**。

## 核心架构链路

`[Antigravity 网页前端 (Leviosa 脚本)] <---> [WebSocket 双向透传] <---> [本地 Taichi 后端服务] <---> [本地伪终端 (PTY)]`

### 1. 前端（数据截获与展示）
由于 Leviosa 是注入在浏览器中的，它拥有当前网页 DOM 的绝对控制权，可以提取到比任何沙盒 API 都丰富的上下文：
- **元数据截获**：获取当前访问的 Project ID 和 Conversation ID，实现上下文精准绑定。
- **对话内容截获**：读取 AI 生成的 Code Blocks 和用户的输入。
- **底层请求截获 (高级)**：通过拦截 `window.fetch`，直接获取 AI 的原始 JSON 数据流。
- **终端 UI 渲染**：利用 `Xterm.js` 在页面挂载一个可隐藏的浮层终端（Overlay Modal）。

### 2. 本地 Taichi 后端（桥接与执行）
- **WebSocket 服务器**：监听前端发来的指令和按键流。
- **PTY 进程管理**：使用 `node-pty` 或类似库，在本地真实环境中衍生出 Bash/Zsh 进程。自动识别前端传来的 Project ID，通过 `cd` 定位到正确的工作目录。

### 3. 神级联动场景 (The "Aha" Moments)
- **AI 喂饭执行**：网页中生成的复杂 Bash 脚本，旁边注入【⚡️终端运行】按钮，点击后通过 WebSocket 传给本地 PTY 执行。
- **错误反向追溯**：本地终端运行 `npm run dev` 报错，终端检测到 Error 输出，通过 WebSocket 反推回网页，前端将报错信息直接注入聊天输入框：“帮我看看这个本地报错怎么解决？”
- **终极工具链**：不需要自己开发文件树，直接在全局终端运行 `broot` 或 `yazi`；不需要开发 Git 面板，直接运行 `lazygit`，获得世界级的全功能体验。

## 总结
通过 WebSocket 建立的这条双向通道，Antigravity 的云端智慧与你本地的物理机器实现了零阻力对接。这是远超修修补补画 UI 的终极形态。
